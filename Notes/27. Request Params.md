# Angular Route Parameters (Request Params / Route Params) - Complete Documentation

# What are Route Parameters?

Route Parameters (often mistakenly called **Request Params** in frontend development) are **dynamic values embedded directly in the URL path**. They are used to identify a specific resource.

For example:

```text
/users/101
```

Here:

* `users` → Route
* `101` → Route Parameter (`id`)

Unlike Query Parameters, Route Parameters are part of the URL path and are usually **required**.

> **Note:** In Angular, the correct term is **Route Parameters** or **URL Parameters**. The term **Request Params** is more commonly used in backend frameworks like Express.js or Spring Boot.

---

# Why do we need Route Parameters?

Route Parameters allow us to create **dynamic routes**.

Instead of creating multiple routes:

```text
/users/1
/users/2
/users/3
/users/4
```

We create one route:

```text
/users/:id
```

Angular automatically replaces `:id` with the actual value.

Benefits:

* Dynamic URLs
* Reusable components
* Clean routing
* SEO-friendly URLs
* RESTful routing

# Creating a Route with Parameters

```ts
import { Routes } from '@angular/router';
import { UserComponent } from './user/user';

export const routes: Routes = [

  {
    path: 'users/:id',
    component: UserComponent
  }

];
```

Here:

```text
:id
```

is a route parameter.

# Reading Route Parameters

Angular provides the `ActivatedRoute` service.

```ts
import { ActivatedRoute } from '@angular/router';

constructor(
  private route: ActivatedRoute
) {}
```

---

# Method 1 - Using Snapshot

Use when the component is recreated every time the route changes.

```ts
ngOnInit() {

  const id =
    this.route.snapshot.paramMap.get('id');

  console.log(id);

}
```

URL:

```text
/users/101
```

Output:

```text
101
```

---

# Method 2 - Using params Observable

Recommended when navigating to the same component with different parameter values.

```ts
ngOnInit() {

  this.route.params.subscribe(params => {

    console.log(params['id']);

  });

}
```

If URL changes:

```text
/users/101
```

↓

```text
/users/102
```

The component updates automatically.

---

# Method 3 - Using paramMap

```ts
this.route.paramMap.subscribe(params => {

  console.log(
    params.get('id')
  );

});
```

Advantages:

* Safer API
* Returns `null` if parameter doesn't exist
* Supports `getAll()` for repeated parameters (rare with route params)

# Route Parameters vs Query Parameters

| Route Parameters           | Query Parameters                 |
| -------------------------- | -------------------------------- |
| Required                   | Optional                         |
| Part of the URL path       | Added after `?`                  |
| Used to identify resources | Used for filtering/configuration |
| `/users/10`                | `/users?page=2`                  |

# Using Route Parameters to Fetch Data

```ts
ngOnInit() {

  const id =
    this.route.snapshot.paramMap.get('id');

  this.productService.getProduct(id)
    .subscribe();

}
```

Flow:

```text
URL
 │
 ▼
Route Parameter
 │
 ▼
Component
 │
 ▼
API Call
 │
 ▼
Display Data
```

# Best Practices

* Use Route Parameters for resource identifiers.
* Use Query Parameters for optional values like filters and pagination.
* Use `paramMap` instead of `params` when possible.
* Use `snapshot` only when parameters won't change during the component's lifetime.
* Use the Observable (`params` or `paramMap`) if the same component stays active while the route parameter changes.

---

# Common Mistakes

## Using Query Parameters Instead of Route Parameters

Wrong:

```text
/products?id=10
```

Better:

```text
/products/10
```

when identifying a specific product.

---

## Using Snapshot When Parameters Can Change

Wrong:

```ts
const id =
this.route.snapshot.paramMap.get('id');
```

If navigating:

```text
/users/1
```

↓

```text
/users/2
```

inside the same component.

Better:

```ts
this.route.paramMap.subscribe(...);
```

---

## Hardcoding URLs

Wrong:

```ts
this.router.navigate([
  '/users/10'
]);
```

Better:

```ts
this.router.navigate([
  '/users',
  userId
]);
```

---

# Route Parameters vs Query Parameters

| Feature  | Route Parameters  | Query Parameters          |
| -------- | ----------------- | ------------------------- |
| Required | Yes (usually)     | No                        |
| Position | URL path          | After `?`                 |
| Example  | `/users/10`       | `/users?page=2`           |
| Purpose  | Identify resource | Filter/configure resource |

# Interview Questions

## What are Route Parameters?

Route Parameters are dynamic values embedded in the URL path that identify a specific resource.

---

## Why do we use Route Parameters?

To create dynamic routes and load different resources using the same component.

---

## How do you define Route Parameters?

```ts
{
  path: 'users/:id',
  component: UserComponent
}
```

---

## How do you navigate with Route Parameters?

```ts
this.router.navigate([
  '/users',
  101
]);
```

or

```html
<a [routerLink]="['/users', 101]">
  User
</a>
```

---

## How do you read Route Parameters?

Using:

* `snapshot.paramMap`
* `params`
* `paramMap`

---

## Difference between `snapshot` and `params`?

`snapshot`

* Reads the value once.
* Does not update automatically.

`params`

* Observable.
* Updates whenever the route parameter changes.

---

## Difference between Route Parameters and Query Parameters?

Route Parameters identify a specific resource.

Query Parameters provide optional information such as filters, sorting, searching, or pagination.

---

## Can a route have multiple parameters?

Yes.

Example:

```text
/users/10/orders/100
```

---

## What is `ActivatedRoute`?

`ActivatedRoute` is an Angular service that provides information about the currently active route, including route parameters, query parameters, fragments, and route data.
