# Signals in Angular

# What is Signal?

A Signal is a reactive primitive introduced in Angular for managing state.

It stores a value and automatically tracks where it is being used.

When the value changes, Angular updates only the places that depend on it.

Think of Signal as:

```text
A smart variable that knows who is using it.
```

Example:

```ts
import { signal } from '@angular/core';

count = signal(0);
```

Reading value:

```ts
count();
```

Updating value:

```ts
count.set(10);
```

Output:

```text
Old: 0
New: 10
```

---

# Why Signals?

Before Signals, Angular mainly used:

- Variables
- RxJS Observables
- Change Detection

Problem:

When one value changes:

```text
Angular checks the entire component tree
```

This creates unnecessary rendering.

Signals solve this.

Benefits:

- Fine-grained reactivity
- Faster rendering
- Less memory usage
- Better performance
- Simpler state management
- Predictable updates

---

# How Signals Work

Signals track dependencies automatically.

Flow:

```text
Signal Created → Value Read → Dependency Registered → Value Updated → Dependent UI Updates
```

Example:

```ts
count = signal(0);

double = computed(() => count() * 2);
```

Dependency graph:

```text
count → double
```

If:

```ts
count.set(5);
```

Angular knows:

```text
Only "double" depends on count
```

So only double updates.

Result:

```text
double = 10
```

---

# Basic Signal Syntax

Creating signal:

```ts
name = signal('Angular');
```

Reading:

```ts
name();
```

Updating:

```ts
name.set('React');
```

Mutating:

```ts
count.update((value) => value + 1);
```

Example:

```ts
count = signal(0);

increment() {
  this.count.update(val => val + 1);
}
```

---

# Types of Signals

Angular provides 3 main signal APIs:

1. Writable Signal
2. Computed Signal
3. Effect

---

# 1. Writable Signal

Stores mutable state.

Example:

```ts
count = signal(0);
```

Read:

```ts
count();
```

Write:

```ts
count.set(5);
```

Update:

```ts
count.update((v) => v + 1);
```

---

# 2. Computed Signal

Derived state.

Read-only.

Automatically recalculates when dependencies change.

Example:

```ts
count = signal(10);

doubleCount = computed(() => count() * 2);
```

Flow:

```text
count → doubleCount
```

If:

```ts
count.set(20);
```

Output:

```text
doubleCount = 40
```

Important:

Cannot manually set computed values.

Wrong:

```ts
doubleCount.set(100);
```

Error.

---

# 3. Effect

Runs side effects when signals change.

Example:

```ts
effect(() => {
  console.log(this.count());
});
```

Whenever count changes:

```text
Effect re-runs automatically
```

Use cases:

- API calls
- Logging
- LocalStorage sync

---

# Fine-Grained Reactivity

This is the biggest feature of Signals.

Normal Angular:

```text
One change → Full component checked
```

Signals:

```text
One change → Only exact dependency updated
```

Example:

```ts
name = signal('Niroj');
age = signal(25);
```

Template:

```html
<p>{{ name() }}</p>
```

If:

```ts
age.set(26);
```

Angular will NOT re-render:

```html
<p>{{ name() }}</p>
```

Because it depends only on `name`.

This is called:

```text
Fine-Grained Reactivity
```

Meaning:

```text
Update only what changed.
```

---

# Signal Graph

Signal Graph is Angular’s dependency map.

Angular internally builds:

```text
Signal → Who depends on it
```

Example:

```ts
count = signal(10);

double = computed(() => count() * 2);

triple = computed(() => count() * 3);
```

Graph:

```text
        count
       /     \
   double    triple
```

If:

```ts
count.set(20);
```

Angular updates:

```text
double = 40
triple = 60
```

Only those.

Not everything.

This dependency network is called Signal Graph.

---

# Avoid Over-Rendering

Old Angular:

```text
State changes → Zone.js → Full Change Detection
```

Signal Angular:

```text
State changes → Exact dependency update
```

Example:

Component:

```ts
name = signal('Niroj');
age = signal(25);
```

Template:

```html
<p>{{ name() }}</p>
```

If age changes:

```ts
age.set(26);
```

No UI re-render for name.

Because name is unrelated.

This avoids:

- Unnecessary DOM checks
- Extra CPU usage
- UI lag

This is how Signals improve performance.

# Signal Methods

## set()

Replace value.

```ts
count.set(100);
```

---

## update()

Update based on previous value.

```ts
count.update((v) => v + 1);
```

---

## mutate() (older pattern / object mutation concept)

Used for object modifications.

Example:

```ts
user = signal({
  name: 'Niroj',
  age: 25,
});

user.update((u) => ({
  ...u,
  age: 26,
}));
```

---

# Signals with Arrays

Example:

```ts
users = signal<string[]>([]);
```

Add item:

```ts
users.update((arr) => [...arr, 'Niroj']);
```

Remove item:

```ts
users.update((arr) => arr.filter((x) => x !== 'Niroj'));
```

---

# Signals with Objects

Example:

```ts
profile = signal({
  name: 'Niroj',
  age: 25,
});
```

Update:

```ts
profile.update((p) => ({
  ...p,
  age: 26,
}));
```

---

# Signals in Template

Example:

```html
<p>{{ count() }}</p>
<button (click)="increment()">+</button>
```

Component:

```ts
count = signal(0);

increment() {
  this.count.update(v => v + 1);
}
```

Output:

```text
0 → 1 → 2 → 3
```

---

# Signal vs Normal Variable

Normal variable:

```ts
count = 0;
```

Signal:

```ts
count = signal(0);
```

Difference:

| Normal Variable | Signal              |
| --------------- | ------------------- |
| No tracking     | Tracks dependencies |
| Manual updates  | Auto reactive       |
| Full CD cycle   | Fine-grained update |
| Less optimized  | Highly optimized    |

---

# Signal vs Observable

| Signal       | Observable      |
| ------------ | --------------- |
| Synchronous  | Async           |
| Pull-based   | Push-based      |
| Simple state | Stream handling |
| Easy to read | More operators  |

Use Signal for:

- Local state
- UI state

Use Observable for:

- API calls
- WebSocket
- Async streams

---

# Best Practices

- Use signal for local state
- Use computed for derived values
- Use effect for side effects
- Avoid too many nested effects
- Keep signals small
- Use immutable updates

---

# Common Mistakes

## Forgetting ()

Wrong:

```ts
count;
```

Correct:

```ts
count();
```

Signal is a function.

---

## Updating computed signal

Wrong:

```ts
double.set(20);
```

Computed is read-only.

---

## Using effect for derived data

Wrong:

```ts
effect(() => {
  this.total = this.count() * 2;
});
```

Correct:

```ts
total = computed(() => this.count() * 2);
```
