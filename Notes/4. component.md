# Angular Component

## 1. What is Component?

A **Component** is the basic building block of an Angular application.

It controls:

- **HTML (View)** → What user sees
- **CSS (Style)** → How it looks
- **TypeScript (Logic)** → How it works

Simple meaning:

A component is a small part of the UI.

### Examples:

- Navbar
- Footer
- Login Form
- Sidebar
- Product Card

Application = Collection of Components

Example:

Navbar + Sidebar + Content + Footer = Full App

---

## 2. Types of Component

There are mainly 2 types:

### 1. Root Component

This is the main component where the Angular application starts.

Example:

```ts
@Component({
  selector: 'app-root'
})
```

Usually `AppComponent` is the root component.

It loads first.

---

### 2. Child / Reusable Component

These components are used inside the root component.

Examples:

- Header Component
- Login Component
- Profile Component

Example:

```ts
@Component({
  selector: 'app-header'
})
```

Used like:

```html
<app-header></app-header>
```

These components can be reused anywhere.

---

## 3. How Component is Built?

A component mainly has 3 files:

### 1. TypeScript File (.ts)

Contains logic.

Example:

```ts
export class AppComponent {
  title = "My Angular App";
}
```

---

### 2. HTML File (.html)

Contains UI.

Example:

```html
<h1>{{ title }}</h1>
```

---

### 3. CSS File (.css)

Contains styles.

Example:

```css
h1 {
  color: blue;
}
```

---

### Full Component Example:

#### user.component.ts

```ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
  styleUrls: ['./user.component.css']
})
export class UserComponent {
  name = "John";
}
```

#### user.component.html

```html
<h1>{{ name }}</h1>
```

#### user.component.css

```css
h1 {
  color: red;
}
```

Output:

```txt
John
```

---

## 4. What is Selector, Template and Styles?

### A) Selector

Selector is the custom HTML tag used to call the component.

Example:

```ts
selector: 'app-user'
```

Usage:

```html
<app-user></app-user>
```

Simple:

Selector = Component name in HTML

---

### B) Template

Template defines what user sees.

Example:

Inline template:

```ts
template: `<h1>Hello</h1>`
```

External template:

```ts
templateUrl: './user.component.html'
```

Simple:

Template = UI Structure

---

### C) Styles

Styles define how the component looks.

Example:

Inline styles:

```ts
styles: [`h1 { color: blue; }`]
```

External styles:

```ts
styleUrls: ['./user.component.css']
```

Simple:

Styles = Design

---

## 5. Interview Questions

### Q1. What is a Component in Angular?

**Answer:**

A component is a class that controls a part of the UI.

---

### Q2. What are the main parts of a component?

**Answer:**

- Selector
- Template
- Styles
- Class

---

### Q3. Difference between `template` and `templateUrl`?

**Answer:**

`template` → HTML written directly inside component.

Example:

```ts
template: `<h1>Hello</h1>`
```

`templateUrl` → HTML written in separate file.

Example:

```ts
templateUrl: './app.component.html'
```

---

### Q4. Difference between `styles` and `styleUrls`?

**Answer:**

`styles` → CSS written directly inside component.

`styleUrls` → CSS written in separate file.

---

### Q5. What is Root Component?

**Answer:**

The first component loaded in Angular application.

Usually:

```ts
AppComponent
```

---

HTML:
