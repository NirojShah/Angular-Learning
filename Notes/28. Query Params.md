# Angular Query Parameters - Complete Documentation

# What are Query Parameters?

Query Parameters are optional key-value pairs that are appended to the end of a URL after the `?` symbol.

They are used to send additional information to a route **without changing the route path**.

Example:

```text
/products?page=1
```

Multiple query parameters:

```text
/products?page=1&limit=10&category=laptop
```

Here:

* `page = 1`
* `limit = 10`
* `category = laptop`

---

# Why Do We Need Query Parameters?

Query parameters are useful when you want to:

* Filter data
* Search data
* Sort records
* Implement pagination
* Pass optional values
* Share URLs with applied filters
* Preserve application state

Examples:

```text
/products?category=mobile
```

```text
/products?category=mobile&brand=apple
```

```text
/products?search=iphone
```

```text
/products?page=2&limit=20
```

Unlike Route Parameters, Query Parameters are **optional**.

---

# Route Parameters vs Query Parameters

## Route Parameters

Route:

```ts
{
  path: 'users/:id',
  component: UserComponent
}
```

URL:

```text
/users/10
```

Reading:

```ts
id = 10
```

---

## Query Parameters

Route:

```ts
{
  path: 'users',
  component: UserComponent
}
```

URL:

```text
/users?page=2
```

Reading:

```ts
page = 2
```

---

# Comparison

| Route Parameters            | Query Parameters                       |
| --------------------------- | -------------------------------------- |
| Required                    | Optional                               |
| Part of the URL path        | Added after `?`                        |
| Used to identify a resource | Used to filter or configure a resource |
| Example: `/users/10`        | Example: `/users?page=2`               |

# How Query Parameters Work

Flow:

```text
Component A
      │
      ▼
Navigate with Query Params
      │
      ▼
Angular Router
      │
      ▼
Browser URL Updates
      │
      ▼
Component B Reads Query Params
```

# Creating Routes

```ts
import { Routes } from '@angular/router';
import { ProductsComponent } from './products/products';

export const routes: Routes = [

  {
    path: 'products',
    component: ProductsComponent
  }

];
```

Notice that the route does **not** define query parameters.

---

# Passing Query Parameters

Using `Router.navigate()`:

```ts
import { Router } from '@angular/router';

constructor(private router: Router) {}

goToProducts() {

  this.router.navigate(
    ['/products'],
    {
      queryParams: {
        page: 1,
        limit: 10,
        category: 'Laptop'
      }
    }
  );

}
```

Generated URL:

```text
/products?page=1&limit=10&category=Laptop
```

---

# Using routerLink

```html
<a
  [routerLink]="['/products']"
  [queryParams]="{
    page: 1,
    category: 'Laptop'
  }">

  Products

</a>
```

Generated URL:

```text
/products?page=1&category=Laptop
```

# Reading Query Parameters

Angular provides `ActivatedRoute`.

```ts
import { ActivatedRoute } from '@angular/router';

constructor(
  private route: ActivatedRoute
) {}
```

---

# Method 1 - Snapshot

Best when query parameters don't change while the component is active.

```ts
ngOnInit() {

  const page =
    this.route.snapshot.queryParamMap.get('page');

  console.log(page);

}
```

Example URL:

```text
/products?page=5
```

Output:

```text
5
```

---

# Method 2 - queryParams Observable

Recommended when query parameters may change without recreating the component.

```ts
ngOnInit() {

  this.route.queryParams.subscribe(params => {

    console.log(params);

  });

}
```

Output:

```text
{
  page: "1",
  category: "Laptop"
}
```

---

# Method 3 - queryParamMap

```ts
this.route.queryParamMap.subscribe(params => {

  console.log(
    params.get('page')
  );

});
```

Advantages:

* Safer API
* Returns `null` if parameter doesn't exist
* Easier to read

---

# Getting Multiple Query Parameters

```ts
this.route.queryParams.subscribe(params => {

  const page = params['page'];
  const limit = params['limit'];
  const category = params['category'];

});
```

---

# Default Values

Sometimes a parameter may not exist.

Example:

```text
/products
```

Use a default value.

```ts
const page =
  this.route.snapshot.queryParamMap.get('page') ?? '1';
```

Output:

```text
1
```

---

# Updating Query Parameters

```ts
this.router.navigate(
  [],
  {
    queryParams: {
      page: 2
    }
  }
);
```

Current route remains the same.

Only the query parameters are updated.

---

# Preserving Existing Query Parameters

Suppose the URL is:

```text
/products?page=2&category=Laptop
```

Now you add:

```ts
this.router.navigate(
  [],
  {
    queryParams: {
      sort: 'price'
    },
    queryParamsHandling: 'merge'
  }
);
```

Result:

```text
/products?page=2&category=Laptop&sort=price
```

Without `merge`:

```text
/products?sort=price
```

Everything else is removed.

---

# queryParamsHandling Options

## merge

Merge old and new query parameters.

```ts
queryParamsHandling: 'merge'
```

---

## preserve

Keep existing query parameters.

```ts
queryParamsHandling: 'preserve'
```

Example:

Current URL:

```text
/products?page=3
```

Navigate:

```ts
this.router.navigate(
  ['/cart'],
  {
    queryParamsHandling: 'preserve'
  }
);
```

Result:

```text
/cart?page=3
```

---

# Removing Query Parameters

Set the value to `null`.

```ts
this.router.navigate(
  [],
  {
    queryParams: {
      page: null
    },
    queryParamsHandling: 'merge'
  }
);
```

Result:

Before:

```text
/products?page=3&category=Laptop
```

After:

```text
/products?category=Laptop
```

```ts
    queryParams: {
```


# Best Practices

* Use query parameters only for optional data.
* Use route parameters for resource identifiers.
* Use `queryParamMap` for safer access.
* Provide default values when parameters are optional.
* Use `merge` when updating filters.
* Keep URLs meaningful and readable.

# Common Mistakes

## Using Route Parameters Instead of Query Parameters

Wrong:

```text
/products/Laptop
```

Better:

```text
/products?category=Laptop
```

---

## Forgetting to Subscribe

Wrong:

```ts
const page =
this.route.snapshot.queryParamMap.get('page');
```

Use `queryParams` or `queryParamMap` if values can change while staying on the same component.

---

## Forgetting queryParams

Wrong:

```ts
this.router.navigate([
  '/products',
  {
    page: 2
  }
]);
```

Correct:

```ts
this.router.navigate(
  ['/products'],
  {
    queryParams: {
      page: 2
    }
  }
);
```

# Interview Questions

## What are Query Parameters?

Query Parameters are optional key-value pairs appended to the URL after `?` to pass additional information without changing the route path.

---

## Difference between Route Parameters and Query Parameters?

| Route Parameters   | Query Parameters              |
| ------------------ | ----------------------------- |
| Required           | Optional                      |
| Identify resources | Filter or configure resources |
| `/users/10`        | `/users?page=2`               |

---

## How do you pass Query Parameters?

Using:

```ts
this.router.navigate(
  ['/products'],
  {
    queryParams: {
      page: 1
    }
  }
);
```

or

```html
<a
  [routerLink]="['/products']"
  [queryParams]="{ page: 1 }">
</a>
```

---

## How do you read Query Parameters?

* `snapshot.queryParamMap`
* `queryParams`
* `queryParamMap`

---

## Difference between snapshot and queryParams?

`snapshot`

* Reads values once.
* Doesn't update automatically.

`queryParams`

* Observable.
* Updates whenever query parameters change.

---

## What is queryParamsHandling?

It controls how Angular handles existing query parameters.

Options:

* `merge`
* `preserve`

---

## What happens if the same query parameter exists multiple times?

Example:

```text
/products?category=Laptop&category=Mobile
```

Angular stores multiple values. You can retrieve them using `queryParamMap.getAll('category')`.

---

## Are Query Parameters required?

No.

They are optional.

---

## When should you use Query Parameters?

* Search
* Filters
* Sorting
* Pagination
* Optional configuration
* Tabs
* Sharing application state
