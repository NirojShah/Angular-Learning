# Signal with Store in Angular

# What is Store?

A Store is a centralized place to manage application state.

Instead of storing state inside multiple components:

```text id="9f9p0k"
Component A → own state
Component B → own state
Component C → own state
```

We can store everything in one place:

```text id="kpwq7g"
Global Store → Shared State
```

This makes state management easier.

Think:

```text id="rw89va"
Store = Global Memory for your app
```

Example:

```text id="l2gx7f"
Cart items
User data
Theme
Counter
Auth data
```

All can live inside Store.

## command to creae store - 

ng g s "/core/store" - this is the folder name where you want to create and the last is the name of the store file.

---

# Why do we need Store?

Without store:

Problems:

* Prop drilling
* Duplicate state
* Hard to sync data
* Difficult debugging
* Hard scaling

Example:

```text id="k9l2qv"
Parent → Child → Grandchild → Great-grandchild
```

Passing data everywhere.

Store solves:

```text id="x8a3qz"
Any component can access shared state directly
```

Benefits:

* Single source of truth
* Easier updates
* Better scalability
* Cleaner architecture

---

# Why not NgRx (for beginners)?

NgRx is powerful, but beginners often struggle because it introduces many concepts at once.

NgRx requires:

* Actions
* Reducers
* Effects
* Selectors
* Store setup

Flow:

```text id="ovkp1d"
Action → Reducer → Store → Selector → Component
```

This is good for large apps.

But for beginners:

Problems:

* Too much boilerplate
* Hard to understand
* Steeper learning curve
* More files

Signal Store is simpler.

Flow:

```text id="e4m6ap"
Component → Store Signal → UI
```

Cleaner.

Easy to learn.

Better for small-medium apps.

---

# What is Signal Store?

Signal Store is a store built using Angular Signals.

It manages global state reactively.

Example:

```text id="ldsxvq"
Signal + Global State = Signal Store
```

It gives:

* Shared state
* Fine-grained reactivity
* Simpler API
* Better performance

Think:

```text id="w1lvfc"
Signal Store = lightweight NgRx
```

# Create CounterStore

Create:

```text id="pbmf67"
counter.store.ts
```

Code:

```ts id="ax7i7x"
import { Injectable, signal, computed } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class CounterStore {
  count = signal(0);

  doubleCount = computed(() => this.count() * 2);

  increment() {
    this.count.update(v => v + 1);
  }

  decrement() {
    this.count.update(v => v - 1);
  }

  reset() {
    this.count.set(0);
  }
}
```

# Use CounterStore in Component

Example:

```ts id="jv4d44"
import { Component } from '@angular/core';
import { CounterStore } from './counter.store';

@Component({
  selector: 'app-counter',
  templateUrl: './counter.component.html'
})
export class CounterComponent {
  constructor(public counterStore: CounterStore) {}
}
```

Template:

```html id="zh9sp7"
<h1>{{ counterStore.count() }}</h1>
<h2>{{ counterStore.doubleCount() }}</h2>

<button (click)="counterStore.increment()">+</button>
<button (click)="counterStore.decrement()">-</button>
<button (click)="counterStore.reset()">Reset</button>
```

Flow:

```text id="l8onv4"
UI → Store Method → Signal Updates → UI Updates
```

# Signal Store Flow

Flow:

```text id="l9bnqp"
Store Signal → Component Reads → Dependency Tracking → Store Updates → Exact UI Update
```

Example:

```text id="qk9z1a"

```

Only dependent templates update.

Fine-grained reactivity.

# Local State vs Store

Local State:

```ts id="dnd1rt"
count = signal(0);
```

Only inside one component.

Store:

```ts id="m0qjlwm"
count = signal(0);
```

Shared across app.

Difference:

| Local Signal    | Store Signal |
| --------------- | ------------ |
| Component-level | Global-level |
| Not shared      | Shared       |
| Temporary       | Centralized  |

---

# Adding Effect inside Store

Example:

```ts id="0g5t1z"
constructor() {
  effect(() => {
    console.log(this.count());
  });
}
```

Whenever count changes:

```text id="bdn6je"
Effect runs automatically
```

Useful for:

* Logging
* LocalStorage
* Analytics

---

# Persisting Store Data

Example:

```ts id="0hsg1u"
constructor() {
  effect(() => {
    localStorage.setItem('count', this.count().toString());
  });
}
```

This saves state.

Restore:

```ts id="8a6jti"
count = signal(Number(localStorage.getItem('count')) || 0);
```

Useful for persistence.

# NgRx vs Signal Store

| Feature          | NgRx       | Signal Store      |
| ---------------- | ---------- | ----------------- |
| Learning curve   | Hard       | Easy              |
| Boilerplate      | High       | Low               |
| Performance      | Good       | Excellent         |
| Setup complexity | High       | Low               |
| Debugging        | Advanced   | Simple            |
| Best for         | Large apps | Small/Medium apps |

# Best Practices

* Keep store small
* Split stores by feature
* Use computed for derived values
* Use effect for side effects
* Avoid too much logic inside components
* Keep store reusable

---

# Common Mistakes

# Storing too much in one store

Bad:

```text id="9o0qcm"
Everything in AppStore
```

Better:

```text id="ftl9cn"
AuthStore
CartStore
CounterStore
ThemeStore
```

---

# Heavy logic inside component

Wrong:

```ts id="32m56k"
count.update(v => v + 1);
```

Better:

```ts id="uc8mim"
counterStore.increment();
```

Store should own logic.

---

# Not using computed

Wrong:

```html id="jlwm8p"
{{ counterStore.count() * 2 }}
```

Better:

```ts id="jlwm82"
doubleCount = computed(...)
```

Cleaner.
