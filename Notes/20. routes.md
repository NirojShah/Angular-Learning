# Angular Routing - Complete Documentation

# What is Routing?

Routing is the mechanism that allows users to navigate between different views or pages in a Single Page Application (SPA) without reloading the browser.

Unlike traditional websites, Angular loads the application once and changes only the required component when navigating.

Example:

```text
/login
/dashboard
/profile
/settings
```

Each URL displays a different component.

---

# Why do we need Routing?

Without routing:

* Entire page reloads
* Poor user experience
* Slow navigation

With routing:

* Fast navigation
* No full page refresh
* Better user experience
* Bookmarkable URLs
* Browser history support

---

# What is a Router?

The Angular Router is responsible for:

* Matching URLs
* Loading components
* Handling navigation
* Managing route guards
* Lazy loading modules/components
* Passing route parameters

Flow:

```text
Browser URL
      │
      ▼
Angular Router
      │
      ▼
Route Match
      │
      ▼
Load Component
      │
      ▼
Display View
```

---

# Routing Terminology

| Term             | Description                                       |
| ---------------- | ------------------------------------------------- |
| Route            | Mapping between URL and Component                 |
| Router           | Angular service that handles navigation           |
| RouterOutlet     | Placeholder where routed components are displayed |
| RouterLink       | Directive used for navigation                     |
| ActivatedRoute   | Provides current route information                |
| Route Parameters | Dynamic values inside URLs                        |
| Query Parameters | Optional values after `?`                         |
| Fragment         | URL anchor (`#section`)                           |
| Guards           | Control navigation                                |
| Resolver         | Fetch data before navigation                      |
| Lazy Loading     | Load routes only when needed                      |

# Creating Routing

Create a new Angular project with routing:

```bash
ng new my-app --routing
```

Or add routing later:

```bash
ng generate module app-routing --flat --module=app
```

For standalone Angular applications:

```bash
ng generate component home
ng generate component about
ng generate component contact
```

---

# Basic Route Configuration

Create routes:

```ts
import { Routes } from '@angular/router';
import { HomeComponent } from './home/home';
import { AboutComponent } from './about/about';

export const routes: Routes = [
  {
    path: '',
    component: HomeComponent
  },
  {
    path: 'about',
    component: AboutComponent
  }
];
```

---

# Providing Routes

Standalone application:

```ts
import { bootstrapApplication } from '@angular/platform-browser';
import { provideRouter } from '@angular/router';

bootstrapApplication(AppComponent, {
  providers: [
    provideRouter(routes)
  ]
});
```

# Programmatic Navigation

Using Router service.

```ts
import { Router } from '@angular/router';

constructor(private router: Router) {}

goHome() {
  this.router.navigate(['/']);
}
```

Navigate with parameters:

```ts
this.router.navigate(['/users', 5]);
```

Using URL string:

```ts
this.router.navigateByUrl('/dashboard');
```

---

# Route Parameters

Dynamic values inside URL.

Route:

```ts
{
  path: 'users/:id',
  component: UserComponent
}
```

Navigate:

```text
/users/101
```

Read parameter:

```ts
import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {}

ngOnInit() {
  console.log(this.route.snapshot.params['id']);
}
```

Reactive approach:

```ts
this.route.params.subscribe(params => {
  console.log(params['id']);
});
```

---

# Query Parameters

Example URL:

```text
/products?page=2&category=laptop
```

Navigation:

```ts
this.router.navigate(['/products'], {
  queryParams: {
    page: 2,
    category: 'laptop'
  }
});
```

Reading:

```ts
this.route.queryParams.subscribe(params => {
  console.log(params);
});
```

---

# Route Guards

Guards control access to routes.

Types:

| Guard            | Purpose                          |
| ---------------- | -------------------------------- |
| CanActivate      | Allow/Block navigation           |
| CanDeactivate    | Prevent leaving page             |
| CanActivateChild | Protect child routes             |
| CanMatch         | Decide whether a route can match |
| Resolve          | Load data before component       |

Example:

```ts
{
  path: 'admin',
  component: AdminComponent,
  canActivate: [authGuard]
}
```

Simple Guard:

```ts
import { CanActivateFn } from '@angular/router';

export const authGuard: CanActivateFn = () => {
  return true;
};
```

---

# Route Resolver

Loads data before navigation.

```ts
{
  path: 'users/:id',
  component: UserComponent,
  resolve: {
    user: userResolver
  }
}
```

Access:

```ts
this.route.data.subscribe(data => {
  console.log(data['user']);
});
```

---

# Lazy Loading

Loads routes only when needed.

Standalone component:

```ts
{
  path: 'admin',
  loadComponent: () =>
    import('./admin/admin')
      .then(c => c.AdminComponent)
}
```

Feature routes:

```ts
{
  path: 'products',
  loadChildren: () =>
    import('./products/products.routes')
      .then(r => r.PRODUCT_ROUTES)
}
```

Benefits:

* Faster initial load
* Better performance
* Smaller bundles

# Router Events

Useful for tracking navigation.

```ts
import {
  Router,
  NavigationStart,
  NavigationEnd
} from '@angular/router';

constructor(router: Router) {

  router.events.subscribe(event => {

    if(event instanceof NavigationStart){
      console.log('Navigation Started');
    }

    if(event instanceof NavigationEnd){
      console.log('Navigation Finished');
    }

  });

}
```

Common Events:

* NavigationStart
* NavigationEnd
* NavigationCancel
* NavigationError

# Scroll Position Restoration

Enable:

```ts

  scrollPositionRestoration: 'enabled'
}))
```

# Preloading Strategy

Preload lazy-loaded routes.

```ts

  routes,
  withPreloading(PreloadAllModules)
);
```

Benefits:

* Faster navigation
* Better user experience

# Best Practices

* Keep routes modular.
* Lazy load feature areas.
* Protect sensitive routes with guards.
* Use route resolvers for initial data.
* Use standalone routing in Angular 17+.
* Keep wildcard route last.
* Avoid deeply nested routes.

# Common Mistakes

## Forgetting RouterOutlet

Wrong:

```html
<h1>App</h1>
```

Correct:

```html
<router-outlet></router-outlet>
```

---

## Wildcard Route First

Wrong:

```ts
{
  path: '**',
  component: NotFoundComponent
}
```

before other routes.

Correct:

Keep it last.

---

## Missing pathMatch

Wrong:

```ts
{
  path: '',
  redirectTo: 'home'
}
```

Correct:

```ts
{
  path: '',
  redirectTo: 'home',
  pathMatch: 'full'
}
```

---

## Using href instead of routerLink

Wrong:

```html
<a href="/about">
```

Correct:

```html
<a routerLink="/about">
```
