# Signals + Template Binding in Angular

# What is Signals + Template Binding?

Signals + Template Binding means using Angular Signals directly inside the HTML template.

Signals can be bound in templates just like normal variables, but with reactivity.

Example:

```ts
count = signal(0);
```

Template:

```html
<p>{{ count() }}</p>
```

Here:

```text
count() → Reads signal value
```

Whenever signal updates:

```text
Template updates automatically
```

This creates reactive UI without manual change detection.

---

# Why use Signals in Templates?

Before Signals:

Angular relied on:

* Normal variables
* Zone.js
* Full component tree checking

Problem:

```text
One variable changes → Whole component re-check
```

With Signals:

```text
One signal changes → Only dependent template updates
```

Benefits:

* Fine-grained updates
* Faster UI rendering
* Less unnecessary DOM work
* Cleaner reactive state
* Better performance

---

# How Signals work inside Template

Flow:

```text
Signal Created → Template Reads Signal → Angular Registers Dependency → Signal Updates → Only That Part Re-renders
```

Example:

Component:

```ts
name = signal("Niroj");
```

Template:

```html
<p>{{ name() }}</p>
```

Dependency graph:

```text
name → paragraph
```

If:

```ts
name.set("Angular");
```

Only this updates:

```html
<p>{{ name() }}</p>
```

Angular knows exactly where `name()` was used.

This is dependency tracking.

# Basic Signal Template Syntax

## Reading Signal

Component:

```ts
count = signal(0);
```

Template:

```html
<p>{{ count() }}</p>
```

Important:

Signal is called like a function.

Wrong:

```html
<p>{{ count }}</p>
```

Correct:

```html
<p>{{ count() }}</p>
```

Because signals are getter functions.

# Signal Binding Types in Template

Signals can be used in multiple Angular bindings.

Main types:

1. Interpolation
2. Property Binding
3. Event Binding
4. Two-Way Style Updates
5. Structural Directives
6. Class Binding
7. Style Binding

---

# 1. Interpolation with Signals

Most common.

Component:

```ts
username = signal("Niroj");
```

Template:

```html
<p>{{ username() }}</p>
```

Output:

```text
Niroj
```

Update:

```ts
username.set("Angular");
```

Output:

```text
Angular
```

---

# 2. Property Binding with Signals

Signals can be used in property bindings.

Component:

```ts
imageUrl = signal("profile.png");
```

Template:

```html
<img [src]="imageUrl()">
```

If:

```ts
imageUrl.set("new-profile.png");
```

Only src updates.

Example:

```html
<button [disabled]="isLoading()">
  Save
</button>
```

Component:

```ts
isLoading = signal(false);
```

---

# 3. Event Binding with Signals

Signals can be updated from events.

Component:

```ts
count = signal(0);

increment() {
  this.count.update(v => v + 1);
}
```

Template:

```html
<button (click)="increment()">+</button>
<p>{{ count() }}</p>
```

Flow:

```text
Click → Signal Updates → Template Updates
```

---

# 4. Two-Way Style Pattern with Signals

Signals don’t directly use ngModel.

Instead:

Component:

```ts
name = signal("");
```

Template:

```html
<input
  [value]="name()"
  (input)="name.set($any($event.target).value)"
>
```

Flow:

```text
Signal → Input Value
Input Change → Signal Update
```

This behaves like two-way binding.

---

# 5. Structural Directives with Signals

Signals work inside:

* @if
* @for
* *ngIf
* *ngFor

---

## Example: Conditional Rendering

Component:

```ts
show = signal(true);
```

Template:

```html
@if (show()) {
  <p>Hello</p>
}
```

If:

```ts
show.set(false);
```

Angular removes the block.

---

## Example: Looping

Component:

```ts
users = signal(["Niroj", "Angular"]);
```

Template:

```html
@for (user of users(); track user) {
  <p>{{ user }}</p>
}
```

If:

```ts
users.update(arr => [...arr, "React"]);
```

Only new item renders.

Not full list.

This is optimized rendering.

---

```html
  User
```

If:

Class removes automatically.

# 7. Style Binding with Signals


Template:

If:

Text color updates.

# Signals with Computed in Template

Computed signals are commonly used in templates.

Component:

```ts
count = signal(10);

double = computed(() => count() * 2);
```

Template:

```html
<p>{{ double() }}</p>
```

If:

```ts
count.set(20);
```

Output:

```text
40
```

Dependency graph:

```text
count → double → template
```

---

# Signals with Effect and Template

Effects don’t render UI directly.

Used for side effects.

Example:

```ts
effect(() => {
  console.log(this.count());
});
```

Template:

```html
<p>{{ count() }}</p>
```

Flow:

```text
Signal Changes → Template Updates + Effect Runs
```

---

# Fine-Grained Template Reactivity

Example:

Component:

```ts
name = signal("Niroj");
age = signal(25);
```

Template:

```html
<p>{{ name() }}</p>
```

If:

```ts
age.set(26);
```

Angular does NOT re-render:

```html
<p>{{ name() }}</p>
```

Because:

```text
Template depends only on name
```

This avoids unnecessary rendering.

This is fine-grained reactivity.

---

# Signal Graph in Template

Example:

```ts
count = signal(5);

double = computed(() => count() * 2);

message = computed(() => `Value: ${double()}`);
```

Template:

```html
<p>{{ message() }}</p>
```

Graph:

```text
count → double → message → template
```

If count changes:

```ts
count.set(10);
```

Angular updates:

```text
double = 20
message = Value: 20
template updates
```

Only dependent nodes update.

---

# Avoid Over-Rendering in Templates

Old Angular:

```text
State Change → Zone.js → Full Change Detection
```

Signals:

```text
State Change → Exact Template Node Update
```

Example:

100 template bindings.

Only 1 signal changes.

Normal Angular:

```text
Checks all 100
```

Signals:

```text
Checks only 1
```

Huge performance gain.

---

# Best Practices

* Always call signals with ()
* Use computed for derived values
* Keep signals small
* Avoid heavy logic inside template
* Use update() for previous-value updates
* Prefer signals for UI state

---

# Common Mistakes

## Forgetting ()

Wrong:

```html
<p>{{ count }}</p>
```

Correct:

```html
<p>{{ count() }}</p>
```

---

## Heavy computations in template

Wrong:

```html
<p>{{ count() * 1000 }}</p>
```

Better:

```ts
bigValue = computed(() => count() * 1000);
```

Template:

```html
<p>{{ bigValue() }}</p>
```

---

## Using effect for UI values

Wrong:

```ts
effect(() => {
  this.total = this.count() * 2;
});
```

Correct:

```ts
total = computed(() => this.count() * 2);
```
