# Event Binding in Angular

## Definition

Event Binding is a technique in Angular used to listen to DOM events (user actions) and execute a method from the component.

It creates communication from **View → Component**.

Example:

* Button click
* Input typing
* Form submission
* Mouse hover
* Keyboard press

---

## Why it is used

Event binding makes the application interactive.

Without event binding:

* Buttons cannot perform actions
* Forms cannot submit
* Inputs cannot react to typing
* Keyboard and mouse actions cannot be captured

It helps Angular respond to user behavior.

---

## How it works

Angular attaches an event listener to the DOM element.

When the event happens:

1. User performs an action
2. Browser triggers the event
3. Angular catches the event
4. Angular calls the component method
5. Method executes
6. Data updates
7. UI re-renders if needed

Flow:

```text
User Action → DOM Event → Angular Listener → Component Method → UI Update
```

---

## Syntax

Basic syntax:

```html
<element (eventName)="methodName()"></element>
```

Example:

```html
<button (click)="showMessage()">Click Me</button>
```

Using event object:

```html
<input (input)="getData($event)">
```

---

## Example 1: Button Click

### HTML

```html
<button (click)="showMessage()">Click Me</button>

<p>{{ message }}</p>
```

### TypeScript

```ts
export class AppComponent {
  message = '';

  showMessage() {
    this.message = 'Button Clicked';
  }
}
```

### Output

Before click:

```text
Click Me
```

After click:

```text
Button Clicked
```

---

## Example 2: Input Event

### HTML

```html
<input (input)="getData($event)" />

<p>{{ value }}</p>
```

### TypeScript

```ts
export class AppComponent {
  value = '';

  getData(event: any) {
    this.value = event.target.value;
  }
}
```

### Output

If user types:

```text
Angular
```

Result:

```text
Angular
```

## Common Events

| Event      | Description                    |
| ---------- | ------------------------------ |
| click      | Triggered when clicked         |
| input      | Triggered while typing         |
| change     | Triggered when value changes   |
| keydown    | Triggered when key is pressed  |
| keyup      | Triggered when key is released |
| submit     | Triggered on form submit       |
| mouseenter | Triggered when mouse enters    |
| mouseleave | Triggered when mouse leaves    |

Example:

```html
<input (keyup)="typing()" />
```

---

## Passing Parameters

You can pass custom parameters.

Example:

```html
<button (click)="save('Angular')">Save</button>
```

```ts
save(name: string) {
  console.log(name);
}
```

Output:

```text
Angular
```

---

## Passing Multiple Parameters

Example:

```html
<button (click)="save('Angular', $event)">Save</button>
```

```ts
save(name: string, event: any) {
  console.log(name);
  console.log(event);
}
```

---

## Important Points

* Event binding uses **parentheses `()`**
* It works from **View → Component**
* `$event` gives access to event details
* The method must exist in the component
* Angular automatically runs change detection after the event

---

## Common Mistakes

### 1. Wrong Syntax

Wrong:

```html
<button click="save()"></button>
```

Correct:

```html
<button (click)="save()"></button>
```

---

### 2. Method Not Defined

Wrong:

```html
<button (click)="unknownMethod()"></button>
```

This throws an error.

---

### 3. Using `change` instead of `input`

Wrong if real-time value needed:

```html
<input (change)="getValue($event)">
```

Correct:

```html
<input (input)="getValue($event)">
```

Reason:

* `change` triggers after focus leaves
* `input` triggers immediately while typing

---

## Difference Between Property Binding and Event Binding

| Property Binding | Event Binding            |
| ---------------- | ------------------------ |
| Component → View | View → Component         |
| Uses `[]`        | Uses `()`                |
| Sends data to UI | Receives actions from UI |

Example:

Property Binding:

```html
<input [value]="name">
```

Event Binding:

```html
<input (input)="getName($event)">
```
