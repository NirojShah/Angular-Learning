# Angular `router.navigate()` - Complete Documentation

# What is `router.navigate()`?

`router.navigate()` is a method provided by Angular's `Router` service that allows you to **navigate programmatically** from one route to another.

Instead of using a clickable link (`routerLink`), you can navigate using TypeScript code.

Example:

```ts
this.router.navigate(['/home']);
```

This changes the URL to:

```text
/home
```

---

# Why do we need `router.navigate()`?

`router.navigate()` is useful when navigation depends on application logic.

Examples:

* After successful login
* After form submission
* Redirect after logout
* Navigate based on user role
* Redirect after API success
* Wizard or multi-step forms
* Conditional navigation

Example:

```text
User Logs In
      │
      ▼
Login Success
      │
      ▼
router.navigate(['/dashboard'])
```

---

# router.navigate() vs routerLink

| `router.navigate()`     | `routerLink`                 |
| ----------------------- | ---------------------------- |
| Used in TypeScript      | Used in HTML Template        |
| Programmatic navigation | User clicks a link           |
| Can include conditions  | Simple navigation            |
| Used after API calls    | Used for menus/buttons/links |

Example:

### routerLink

```html
<a routerLink="/dashboard">
  Dashboard
</a>
```

---

### router.navigate()

```ts
login() {

  if (success) {
    this.router.navigate(['/dashboard']);
  }

}
```

# Navigate with Route Parameters

Route:

```ts
{
  path: 'users/:id',
  component: UserComponent
}
```

Navigate:

```ts
this.router.navigate([
  '/users',
  101
]);
```

Generated URL:

```text
/users/101
```

---

# Navigate with Multiple Route Parameters

Route:

```ts
{
  path: 'users/:userId/orders/:orderId',
  component: OrderComponent
}
```

Navigate:

```ts
this.router.navigate([
  '/users',
  10,
  'orders',
  200
]);
```

Generated URL:

```text
/users/10/orders/200
```

# Navigate with Query Parameters

```ts
this.router.navigate(
  ['/products'],
  {
    queryParams: {
      page: 1,
      category: 'Laptop'
    }
  }
);
```

Generated URL:

```text
/products?page=1&category=Laptop
```

# Passing Multiple Query Parameters

```ts
this.router.navigate(
  ['/products'],
  {
    queryParams: {
      page: 2,
      limit: 10,
      sort: 'price'
    }
  }
);
```

Result:

```text
/products?page=2&limit=10&sort=price
```

---

# Merging Existing Query Parameters

Current URL:

```text
/products?page=2
```

Navigate:

```ts
this.router.navigate(
  [],
  {
    queryParams: {
      sort: 'price'
    },
    queryParamsHandling: 'merge'
  }
);
```

Result:

```text
/products?page=2&sort=price
```

---

# Navigate Relative to Current Route

Current URL:

```text
/users
```

Navigate:

```ts
constructor(
  private router: Router,
  private route: ActivatedRoute
) {}

this.router.navigate(
  ['profile'],
  {
    relativeTo: this.route
  }
);
```

Generated URL:

```text
/users/profile
```

# Replace Browser History

Normally:

```ts
this.router.navigate(['/dashboard']);
```

adds a new browser history entry.

Use `replaceUrl`:

```ts
this.router.navigate(
  ['/dashboard'],
  {
    replaceUrl: true
  }
);
```

Useful after login or logout to prevent navigating back to the login page.

---

# Skip Browser History

```ts
this.router.navigate(
  ['/dashboard'],
  {
    skipLocationChange: true
  }
);
```

The component changes, but the browser URL does not visibly update.

Use only for special cases.

---

# Passing State

```ts
this.router.navigate(
  ['/payment'],
  {
    state: {
      orderId: 500
    }
  }
);
```

Read in the destination component:

```ts
const data = history.state;

console.log(data.orderId);
```

> State is not stored in the URL and is lost after a full page refresh.

# Common Navigation Options

| Option                | Purpose                               |
| --------------------- | ------------------------------------- |
| `queryParams`         | Pass query parameters                 |
| `queryParamsHandling` | Merge or preserve query parameters    |
| `relativeTo`          | Navigate relative to current route    |
| `replaceUrl`          | Replace browser history entry         |
| `skipLocationChange`  | Navigate without changing visible URL |
| `state`               | Pass temporary navigation state       |

---

# router.navigate() vs router.navigateByUrl()

## router.navigate()

Uses route segments.

```ts
this.router.navigate([
  '/users',
  101
]);
```

---

## router.navigateByUrl()

Uses a complete URL string.

```ts
this.router.navigateByUrl(
  '/users/101'
);
```

---

# Comparison

| `navigate()`               | `navigateByUrl()`      |
| -------------------------- | ---------------------- |
| Array of route segments    | Complete URL string    |
| Easier for dynamic values  | Simpler for fixed URLs |
| Supports route composition | Good for known URLs    |

---

# Best Practices

* Use `routerLink` for simple template navigation.
* Use `router.navigate()` for navigation based on business logic.
* Use route parameters for resource identifiers.
* Use query parameters for optional filters.
* Use `replaceUrl` after login/logout when appropriate.
* Avoid hardcoding URL strings when dynamic route segments can be passed as separate array elements.

---

# Common Mistakes

## Hardcoding Dynamic URLs

Wrong:

```ts
this.router.navigate([
  '/users/10'
]);
```

Better:

```ts
this.router.navigate([
  '/users',
  10
]);
```

---

## Using Query Parameters for Resource IDs

Wrong:

```text
/users?id=10
```

Better:

```text
/users/10
```

---

## Forgetting to Inject Router

Wrong:

```ts
this.router.navigate(['/home']);
```

without injecting:

```ts
constructor(
  private router: Router
) {}
```

# Interview Questions

## What is `router.navigate()`?

`router.navigate()` is a method of Angular's `Router` service used for programmatic navigation between routes.

---

## Difference between `router.navigate()` and `routerLink`?

`router.navigate()` is used in TypeScript code for programmatic navigation, while `routerLink` is used in HTML templates for declarative navigation.

---

## Difference between `navigate()` and `navigateByUrl()`?

`navigate()` accepts an array of route segments.

`navigateByUrl()` accepts a complete URL string.

---

## How do you pass Route Parameters?

```ts
this.router.navigate([
  '/users',
  101
]);
```

---

## How do you pass Query Parameters?

```ts
this.router.navigate(
  ['/products'],
  {
    queryParams: {
      page: 1
    }
  }
);
```

---

## What is `relativeTo`?

`relativeTo` allows navigation relative to the currently activated route instead of using an absolute path.

---

## What does `replaceUrl` do?

It replaces the current browser history entry instead of adding a new one.

---

## What is `skipLocationChange`?

It navigates to another component without changing the visible browser URL.

---

## What is the `state` option?

It passes temporary data during navigation without exposing it in the URL.
