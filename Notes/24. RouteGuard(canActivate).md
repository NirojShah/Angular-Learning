# Custom Reusable Component in Angular

# What is a Reusable Component?

A reusable component is a component that is designed once and can be used multiple times throughout an application.

Instead of writing the same HTML and logic repeatedly, we create a reusable component and pass different data whenever we use it.

Think of it like a function.

```text id="8jxg8u"
Function → Reusable logic

Component → Reusable UI
```

Example:

Instead of creating the same card 20 times:

```text id="mjlwm1"
Product Card
Product Card
Product Card
Product Card
```

We create:

```text id="jlwm2"
<Card />
```

and reuse it everywhere.

---

# Why do we need Reusable Components?

Without reusable components:

* Duplicate code
* Difficult maintenance
* Hard to update UI
* More bugs
* Poor scalability

With reusable components:

* Write once
* Use anywhere
* Easy maintenance
* Better readability
* Better scalability
* Consistent UI

---

# Benefits of Reusable Components

* Reduces code duplication
* Easy to maintain
* Better separation of concerns
* Easier testing
* Improves team productivity
* Consistent design

# Creating a Reusable Component

Generate a component using Angular CLI.

```bash id="jlwm3"
ng generate component shared/card
```

Short command:

```bash id="jlwm4"
ng g c shared/card
```

Generated structure:

```text id="jlwm5"
shared/
 └── card/
      ├── card.ts
      ├── card.html
      ├── card.css
```

---

# Example: Simple Card Component

Card Component:

```ts id="jlwm6"
import { Component } from '@angular/core';

@Component({
  selector: 'app-card',
  templateUrl: './card.html',
  styleUrls: ['./card.css']
})
export class CardComponent {}
```

Template:

```html id="jlwm7"
<div class="card">
  Card Component
</div>
```

Using:

```html id="jlwm8"
<app-card></app-card>
```

Output:

```text id="jlwm9"
Card Component
```

---

# Making the Card Dynamic using input()

Instead of hardcoding values, pass data from the parent.

Child Component:

```ts id="jlwm10"
import { Component, input } from '@angular/core';

@Component({
  selector: 'app-card',
  templateUrl: './card.html'
})
export class CardComponent {

  title = input('');
  description = input('');
  image = input('');

}
```

Template:

```html id="jlwm11"
<div class="card">

  <img [src]="image()" alt="Card Image">

  <h2>{{ title() }}</h2>

  <p>{{ description() }}</p>

</div>
```

# What is ng-content?

`<ng-content>` is Angular's content projection feature.

It allows a parent component to inject custom HTML into a child component.

Instead of passing only data, we can pass complete HTML.

Think of it as a placeholder.

```text id="’wini18"
Parent HTML
      │
      ▼
<ng-content>
      │
      ▼
Rendered inside Child
```

---

# Why use ng-content?

Without `ng-content`:

The child component controls all the HTML.

With `ng-content`:

The parent decides what content to display inside the child.

Benefits:

* Flexible UI
* Reusable layouts
* Better customization
* Cleaner components

---

# Using ng-content

Child Component:

```html id="’wini19"
<div class="card">

  <h2>{{ title() }}</h2>

  <hr>

  <ng-content></ng-content>

</div>
```

---

Parent Component:

```html id="’wini20"
<app-card [title]="'Angular'">

  <p>Angular is a frontend framework.</p>

  <button>Read More</button>

</app-card>
```

Output:

```text id="’wini21"
Angular

Angular is a frontend framework.

[Read More]
```

The button and paragraph come from the parent.

# Multiple Content Projection

Angular supports multiple content slots.

Child:

```html id="’wini22"
<div class="card">

  <header>
    <ng-content select="[card-header]"></ng-content>
  </header>

  <main>
    <ng-content></ng-content>
  </main>

  <footer>
    <ng-content select="[card-footer]"></ng-content>
  </footer>

</div>
```

Parent:

```html id="’wini23"
<app-card>

  <h2 card-header>Angular</h2>

  <p>Signals are reactive.</p>

  <button card-footer>
    Read More
  </button>

</app-card>
```

Output:

```text id="’wini24"
Header

Body

Footer
```

# Best Practices

* Keep components focused on one responsibility.
* Use `input()` for dynamic data.
* Use `ng-content` for flexible layouts.
* Keep styling inside the reusable component.
* Avoid business logic inside reusable UI components.
* Make components configurable instead of hardcoding values.

---

# Common Mistakes

## Hardcoding data

Wrong:

```html id="’wini34"
<h2>Angular</h2>
```

Better:

```html id="’wini35"
<h2>{{ title() }}</h2>
```

---

## Forgetting to call signal inputs

Wrong:

```html id="’wini36"
{{ title }}
```

Correct:

```html id="’wini37"
{{ title() }}
```

---

## Not using ng-content when flexibility is needed

Wrong:

```html id="’wini38"
<button>Read More</button>
```

Better:

```html id="’wini39"
<ng-content></ng-content>
```

Now the parent decides what to display.

---

# input() vs ng-content

| input()                         | ng-content         |
| ------------------------------- | ------------------ |
| Passes data                     | Passes HTML        |
| String, Number, Object, Boolean | Any HTML content   |
| Fixed structure                 | Flexible structure |
| Good for properties             | Good for layouts   |

# Interview Questions

## What is a reusable component?

A component that is created once and reused in multiple places.

---

## Why should we create reusable components?

To reduce duplicate code, improve maintainability, and build scalable applications.

---

## What is `input()`?

A signal-based API used to receive data from a parent component.

---

## What is `ng-content`?

A content projection feature that allows a parent component to inject HTML into a child component.

---

## Difference between `input()` and `ng-content`?

`input()` passes data, while `ng-content` projects HTML content.

---

## Can we use both `input()` and `ng-content` together?

Yes.

This is one of the most common Angular design patterns.

---

## What is content projection?

The process of inserting parent HTML into a child component using `<ng-content>`.

---

## Can `ng-content` have multiple slots?

Yes.

Using the `select` attribute.

---

## Where should reusable components be placed?

Usually inside a `shared` folder.
