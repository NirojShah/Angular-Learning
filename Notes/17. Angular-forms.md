# Forms with Signals in Angular

# What are Forms in Angular?

Forms are used to collect input from users.

Examples:

- Login Form
- Registration Form
- Contact Form
- Profile Form
- Payment Form

Angular provides two types of forms:

1. Template-driven Forms
2. Reactive Forms

With Angular Signals, form state becomes more reactive and easier to manage.

---

# Why use Signals with Forms?

Traditionally, Angular forms rely on:

- `ngModel`
- `FormControl`
- `FormGroup`
- RxJS Observables (`valueChanges`)

Problems:

- More subscriptions
- More boilerplate
- Manual synchronization
- Harder state management

Signals solve these problems.

Benefits:

- Reactive state management
- Automatic UI updates
- Fine-grained reactivity
- Less boilerplate
- Easier to derive values using `computed()`
- Easier side effects using `effect()`

Flow:

```text id="q8zt1d"
User Input → Signal Updates → UI Updates Automatically
```

---

# Types of Forms with Signals

There are two common approaches:

1. Template-driven Forms + Signals
2. Reactive Forms + Signals (Recommended)

---

# 1. Template-driven Forms + Signals

Template-driven forms use `ngModel`.

Signals can be used as the state.

## Example

Component:

```ts id="r6g3wq"
import { Component, signal } from '@angular/core';

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
})
export class UserComponent {
  username = signal('');

  updateName(event: Event) {
    const input = event.target as HTMLInputElement;
    this.username.set(input.value);
  }
}
```

Template:

```html id="vjlwmz"
<input [ngModel]="username()" (ngModelChange)="username.set($event)" />

<p>{{ username() }}</p>
```

Flow:

```text id="0n2a3h"
Input
   │
   ▼
ngModelChange
   │
   ▼
Signal
   │
   ▼
Template Updates
```

---

# Using Computed with Template Forms

Component:

```ts id="dnwvpb"
import { signal, computed } from '@angular/core';

firstName = signal('');
lastName = signal('');

fullName = computed(() => `${this.firstName()} ${this.lastName()}`);
```

Template:

```html id="03vzkb"
<input [ngModel]="firstName()" (ngModelChange)="firstName.set($event)" />

<input [ngModel]="lastName()" (ngModelChange)="lastName.set($event)" />

<p>{{ fullName() }}</p>
```

Whenever either signal changes:

```text id="mjlwmu"
fullName recalculates automatically.
```

---

# Using Effect with Template Forms

```ts id="i4c5dx"
import { effect } from '@angular/core';

constructor() {
  effect(() => {
    console.log(this.username());
  });
}
```

Every input change automatically triggers the effect.

Useful for:

- Logging
- Analytics
- Saving drafts
- Local storage

---

# Limitations of Template-driven Forms

Template-driven forms work well for:

- Small forms
- Simple applications

But become difficult for:

- Complex validation
- Dynamic forms
- Nested forms
- Enterprise applications

Because of this, Angular recommends Reactive Forms for larger projects.

---

# 2. Reactive Forms + Signals (Recommended)

Reactive Forms are the preferred approach when using Signals.

Why?

Reactive Forms already provide structured state management.

Signals make them even better.

Benefits:

- Better scalability
- Easier validation
- Better architecture
- Fine-grained UI updates
- Easier derived state

---

```ts id="1jjlwm"
  name: new FormControl(''),
  email: new FormControl(''),
});
```

Template:

```html id="jlwm07"
<form [formGroup]="userForm">
  <input formControlName="name" />

  <input formControlName="email" />
</form>
```

# Converting Form Values into Signals

Reactive Forms expose:

```ts id="jlwm11"
valueChanges;
```

which is an Observable.

Angular provides:

```ts id="jlwm18"
toSignal();
```

to convert Observables into Signals.

Example:

```ts id="jlwm29"
import { toSignal } from '@angular/core/rxjs-interop';

nameSignal = toSignal(this.userForm.controls.name.valueChanges, {
  initialValue: '',
});
```

Now:

```ts id="jlwm32"
nameSignal();
```

returns the latest value reactively.

---

# Complete Example

Component:

```ts id="jlwm44"
import { Component, computed } from '@angular/core';
import { FormControl, FormGroup } from '@angular/forms';
import { toSignal } from '@angular/core/rxjs-interop';

@Component({
  selector: 'app-user',
  templateUrl: './user.component.html',
})
export class UserComponent {
  userForm = new FormGroup({
    firstName: new FormControl(''),
    lastName: new FormControl(''),
  });

  firstName = toSignal(this.userForm.controls.firstName.valueChanges, { initialValue: '' });

  lastName = toSignal(this.userForm.controls.lastName.valueChanges, { initialValue: '' });

  fullName = computed(() => `${this.firstName()} ${this.lastName()}`);
}
```

Template:

```html id="jlwm51"
<form [formGroup]="userForm">
  <input formControlName="firstName" />

  <input formControlName="lastName" />
</form>

<p>{{ fullName() }}</p>
```

Flow:

```text id="jlwm53"
FormControl
      │
      ▼
valueChanges
      │
      ▼
toSignal()
      │
      ▼
Computed Signal
      │
      ▼
Template
```

# Form Validation with Signals

Example:

```ts id="jlwm71"
email = toSignal(this.userForm.controls.email.valueChanges, {
  initialValue: '',
});

isValidEmail = computed(() => this.email().includes('@'));
```

Template:

```html id="jlwm82"
@if (!isValidEmail()) {
<p>Invalid Email</p>
}
```

No subscriptions required.

# Comparing Template-driven Forms and Reactive Forms with Signals

| Feature        | Template-driven + Signals | Reactive Forms + Signals |
| -------------- | ------------------------- | ------------------------ |
| Learning Curve | Easy                      | Moderate                 |
| Best For       | Small forms               | Large forms              |
| Validation     | Basic                     | Advanced                 |
| Dynamic Forms  | Difficult                 | Easy                     |
| Scalability    | Medium                    | Excellent                |
| Recommended    | Small projects            | Yes                      |

---

# Best Practices

- Prefer Reactive Forms for medium and large applications.
- Use Signals for UI state.
- Use `computed()` for derived values.
- Use `effect()` for side effects.
- Convert `valueChanges` to Signals using `toSignal()`.
- Avoid manual subscriptions whenever possible.

---

# Common Mistakes

## Forgetting initialValue in toSignal()

Wrong:

```ts id="jlwm91"
toSignal(control.valueChanges);
```

Correct:

```ts id="jlwm94"
toSignal(control.valueChanges, {
  initialValue: '',
});
```

---

## Using subscriptions unnecessarily

Wrong:

```ts id="jlwm95"
control.valueChanges.subscribe(...);
```

Better:

```ts id="jlwm96"
const value = toSignal(control.valueChanges);
```

---

## Using computed for side effects

Wrong:

```ts id="jlwm97"
computed(() => {
  console.log(this.name());
});
```

Correct:

```ts id="jlwm98"
effect(() => {
  console.log(this.name());
});
```

---

# When should you use Signals with Forms?

Use Signals when:

- You need reactive UI updates.
- You want derived values.
- You want less RxJS boilerplate.
- You want fine-grained rendering.
- You're building modern Angular applications.

Reactive Forms.

---

## Why is `toSignal()` useful?

It converts an Observable into a Signal.

---

## Can computed() be used with form values?

Yes.

Very common for derived form state.

---

## Can effect() be used with forms?

Yes.

Useful for logging, local storage, analytics, and API calls.

---

## Should we replace Reactive Forms with Signals?

No.

Signals complement Reactive Forms; they do not replace them.

---

## Signal vs valueChanges.subscribe()

| valueChanges.subscribe() | Signal             |
| ------------------------ | ------------------ |
| Manual subscription      | Automatic          |
| Cleanup required         | Managed by Angular |
| Observable               | Signal             |
| More boilerplate         | Cleaner code       |
