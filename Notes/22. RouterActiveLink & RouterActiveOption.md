# Angular `routerLinkActiveOptions`

`routerLinkActiveOptions` is used with Angular's `routerLinkActive` directive to control **how Angular determines whether a route is active**.

By default, Angular performs **partial (prefix) matching**, which means a parent route is also considered active when you're on one of its child routes.

---

# Syntax

```html
<a
  routerLink="/home"
  routerLinkActive="active"
  [routerLinkActiveOptions]="{ exact: true }">
  Home
</a>
```

---

# Why do we need it?

Suppose your application has the following routes:

```
/
├── home
├── products
│   ├── 1
│   ├── 2
│   └── reviews
└── contact
```

Your navigation menu:

```html
<a routerLink="/products" routerLinkActive="active">
    Products
</a>

<a routerLink="/products/1" routerLinkActive="active">
    Product 1
</a>
```

Current URL:

```
/products/1
```

### Without `routerLinkActiveOptions`

Angular performs **prefix matching**.

Since `/products` is the beginning of `/products/1`, **both links become active**.

```
Products   <-- Active
Product 1  <-- Active
```

Sometimes this is useful, but often you want only the exact page to be highlighted.

---

# Using `exact: true`

```html
<a
    routerLink="/products"
    routerLinkActive="active"
    [routerLinkActiveOptions]="{ exact: true }">
    Products
</a>
```

Now Angular only marks the link as active when the URL matches exactly.

| Current URL | Products Active? |
|-------------|------------------|
| `/products` | ✅ Yes |
| `/products/1` | ❌ No |
| `/products/reviews` | ❌ No |

---

# Using `exact: false`

```html
<a
    routerLink="/products"
    routerLinkActive="active"
    [routerLinkActiveOptions]="{ exact: false }">
    Products
</a>
```

This is Angular's default behavior.

| Current URL | Products Active? |
|-------------|------------------|
| `/products` | ✅ Yes |
| `/products/1` | ✅ Yes |
| `/products/reviews` | ✅ Yes |

---

# When should you use it?

## Use `exact: true`

Use it for **top-level navigation** where only the current page should be highlighted.

Example:

```html
<nav>
    <a
        routerLink="/home"
        routerLinkActive="active"
        [routerLinkActiveOptions]="{ exact: true }">
        Home
    </a>

    <a
        routerLink="/about"
        routerLinkActive="active"
        [routerLinkActiveOptions]="{ exact: true }">
        About
    </a>

    <a
        routerLink="/contact"
        routerLinkActive="active"
        [routerLinkActiveOptions]="{ exact: true }">
        Contact
    </a>
</nav>
```

If the current URL is:

```
/home/profile
```

The **Home** link will **not** remain active because the URL is not an exact match.

---

## Use `exact: false`

Use it when you want a **parent section** to remain highlighted while the user navigates through its child pages.

Example route structure:

```
Dashboard
├── Users
├── Reports
└── Settings
```

Navigation:

```html
<a
    routerLink="/dashboard"
    routerLinkActive="active"
    [routerLinkActiveOptions]="{ exact: false }">
    Dashboard
</a>
```

If the current URL is:

```
/dashboard/users
```

Result:

```
Dashboard <-- Active
```

This tells the user they are still inside the Dashboard section.

---

# Real-world Example

```html
<nav>

    <a
        routerLink="/"
        routerLinkActive="active"
        [routerLinkActiveOptions]="{ exact: true }">
        Home
    </a>

    <a
        routerLink="/products"
        routerLinkActive="active">
        Products
    </a>

    <a
        routerLink="/contact"
        routerLinkActive="active"
        [routerLinkActiveOptions]="{ exact: true }">
        Contact
    </a>

</nav>
```

Current URL:

```
/products/42
```

Result:

```
Home      ❌
Products  ✅
Contact   ❌
```

The **Products** link remains active because it represents the parent section.

---

# Summary

| Option | Matching Behavior | Use Case |
|---------|-------------------|----------|
| `{ exact: true }` | Matches only the exact URL. | Home, About, Contact, Login pages. |
| `{ exact: false }` | Matches the current URL and all child URLs. | Dashboard, Products, Admin, Settings sections. |

---

# Key Takeaways

- `routerLinkActive` adds a CSS class when a route is active.
- `routerLinkActiveOptions` controls **how Angular checks if a route is active**.
- `exact: true` requires an exact URL match.
- `exact: false` (default) uses prefix matching.
- Use `exact: true` for standalone pages.
- Use `exact: false` for parent routes that contain child routes.

---

# Interview Question

### Why do we use `routerLinkActiveOptions`?

**Answer:**

`routerLinkActiveOptions` allows us to customize how Angular determines whether a route is active. By default, Angular uses partial (prefix) matching, so a parent route is considered active when any of its child routes are active. Setting `{ exact: true }` changes this behavior so the active class is applied only when the current URL exactly matches the route. This is commonly used in navigation menus to ensure only the correct menu item is highlighted.
