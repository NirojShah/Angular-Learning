# Datatypes in Angular (TypeScript)

## Introduction

Angular uses **TypeScript**, and TypeScript is a strongly typed language.

That means we can define what kind of value a variable should store.

Example:

```ts
let name: string = "Niroj";
let age: number = 25;
let isActive: boolean = true;
```

Here:

* `name` can only store string
* `age` can only store number
* `isActive` can only store boolean

---

## Why we need to define type?

Defining type helps Angular and TypeScript understand what kind of data we are working with.

Benefits:

### 1. Prevents wrong data assignment

```ts
let age: number = 20;
age = "hello"; // Error
```

Without type:

```ts
let age = 20;
age = "hello"; // Can create issues
```

---

### 2. Better code readability

```ts
let username: string;
```

Anyone reading the code knows it stores text.

---

### 3. Better auto-suggestions

Editors like VS Code provide better IntelliSense.

Example:

```ts
let user: string = "Angular";
user.toUpperCase();
```

It will suggest string methods.

---

### 4. Catches bugs early

Errors are found during development, not runtime.

---

## Define type or property of variable?

Syntax:

```ts
variableName: datatype = value;
```

Example:

```ts
name: string = "Angular";
price: number = 100;
isLoggedIn: boolean = false;
```

Inside Angular component:

```ts
export class AppComponent {
  title: string = "My App";
  count: number = 0;
}
```

---

## Common Datatypes in Angular

### String

Stores text.

```ts
let username: string = "Niroj";
```

---

### Number

Stores numbers.

```ts
let age: number = 25;
```

---

### Boolean

Stores true/false.

```ts
let isAdmin: boolean = true;
```

---

### Any

Can store anything.

```ts
let data: any = "Hello";
data = 100;
data = true;
```

Use less because it removes type safety.

---

### Array

Stores multiple values.

```ts
let users: string[] = ["A", "B", "C"];
```

Number array:

```ts
let marks: number[] = [10, 20, 30];
```

---

### Object

Stores key-value pairs.

```ts
let user: object = {
  name: "Niroj",
  age: 25
};
```

Better:

```ts
let user: {name: string, age: number} = {
  name: "Niroj",
  age: 25
};
```

---

### Null

Represents empty value.

```ts
let data: null = null;
```

---

### Undefined

Represents uninitialized.

```ts
let value: undefined = undefined;
```

---

## Define multiple type to property?

Sometimes one variable can store multiple types.

This is called **Union Type**.

Syntax:

```ts
let variable: string | number;
```

Example:

```ts
let id: string | number;

id = 101;
id = "EMP101";
```

Another example:

```ts
status: boolean | string = true;
```

Valid:

```ts
status = "active";
status = false;
```

---

## Define type for functional param?

Function parameters can also have types.

Syntax:

```ts
functionName(param: datatype)
```

Example:

```ts
getUser(name: string) {
  console.log(name);
}
```

Call:

```ts
getUser("Niroj");
```

---

Multiple parameters:

```ts
add(a: number, b: number) {
  return a + b;
}
```

Call:

```ts
add(10, 20);
```

---

Function return type:

Syntax:

```ts
functionName(): datatype
```

Example:

```ts
sum(a: number, b: number): number {
  return a + b;
}
```

---

## Call function with param on button click?

In Angular HTML, we can pass values directly.

Example:

### HTML

```html
<button (click)="getName('Niroj')">Send</button>
```

### TypeScript

```ts
getName(name: string) {
  console.log(name);
}
```

Output:

```text
Niroj
```

---

Passing multiple params:

### HTML

```html
<button (click)="add(10,20)">Add</button>
```

### TypeScript

```ts
add(a: number, b: number) {
  console.log(a + b);
}
```

Output:

```text
30
```

---

Passing variable:

### HTML

```html
<button (click)="showName(username)">Show</button>
```

### TypeScript

```ts
username: string = "Angular";

showName(name: string) {
  console.log(name);
}
```

Output:

```text
Angular
```

---

## Important Points

* Angular uses TypeScript
* TypeScript provides type safety
* Use proper datatypes for better code quality
* Function parameters should have types
* Multiple datatypes can be handled using union `|`
* Avoid `any` unless necessary

---

## Common Mistakes

### 1. Assigning wrong type

Wrong:

```ts
let age: number = "25";
```

Correct:

```ts
let age: number = 25;
```

---

### 2. Using `any` everywhere

Wrong:

```ts
let user: any = "Hello";
```

Better:

```ts
let user: string = "Hello";
```

---

### 3. Missing parameter type

Wrong:

```ts
getName(name) {}
```

Correct:

```ts
getName(name: string) {}
```

## Interview Questions

### What is TypeScript in Angular?

TypeScript is a typed superset of JavaScript used by Angular.

---

### Why do we define datatypes?

To improve safety, readability, and catch errors early.

---

### What is union type?

A variable storing multiple possible datatypes.

Example:

```ts
let id: string | number;
```

---

### What is `any` type?

It allows any datatype but removes type checking.

---

### Difference between `any` and `unknown`?

* `any` allows anything without checks
* `unknown` requires checking before use

---

### How to define array type?

```ts
let names: string[] = [];
```

---

### How to define function parameter type?

```ts
getData(name: string)
```
