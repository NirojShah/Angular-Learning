# Two-Way Binding in Angular

# What is Two-Way Binding?

Two-Way Binding means data flows in both directions:

```text
Component ↔ View
```

It combines:

1. **Property Binding** (Component → View)
2. **Event Binding** (View → Component)

This means:

* If component data changes → UI updates
* If UI data changes → component updates

Example:

```html
<input [(ngModel)]="name">
```

Flow:

```text
Component Value → Input Field
Input Field Change → Component Value
```

This creates synchronization.

---

# Why do we need Two-Way Binding?

Without two-way binding:

You have to manually do:

```html
<input [value]="name" (input)="name = $event.target.value">
```

This is longer and repetitive.

With Two-Way Binding:

```html
<input [(ngModel)]="name">
```

Cleaner and easier.

Benefits:

* Less code
* Real-time sync
* Easy form handling
* Better readability
* Better user interaction

---

# How Two-Way Binding Works

Angular internally combines:

Property Binding:

```html
[value]="name"
```

Event Binding:

```html
(input)="name = $event.target.value"
```

Combined:

```html
[(ngModel)]="name"
```

Flow:

```text
Component → View → User Changes → Component
```

Example:

```ts
name = "Niroj";
```

Template:

```html
<input [(ngModel)]="name">
<p>{{ name }}</p>
```

If user types:

```text
Angular
```

Component automatically updates:

```text
name = "Angular"
```

And UI updates instantly.

---

# Syntax

Basic syntax:

```html
[(ngModel)]="propertyName"
```

Example:

```html
<input [(ngModel)]="username">
```

This special syntax is called:

```text
Banana in a Box
```

Because:

```text
[( )]
```

Looks like a banana inside a box.

# Types of Two-Way Binding in Angular

Angular mainly supports two-way binding in these ways:

1. ngModel (Template-driven forms)
2. Custom Two-Way Binding
3. Signals with Model Inputs (Modern Angular)

---

# 1. ngModel (Most Common)

Used for form inputs.

Before using:

Import FormsModule.

App module:

```ts
import { FormsModule } from '@angular/forms';

@NgModule({
  imports: [FormsModule]
})
export class AppModule {}
```

# 2. Custom Two-Way Binding

Angular allows custom two-way binding between parent and child.

Rule:

To create custom two-way binding:

Need:

```text
@Input()
@Output()
```

Naming convention:

```text
propertyName
propertyNameChange
```

---

## Child Component

```ts
import { Component, Input, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-child',
  template: `
    <input [value]="value" (input)="updateValue($event)">
  `
})
export class ChildComponent {
  @Input() value!: string;

  @Output() valueChange = new EventEmitter<string>();

  updateValue(event: any) {
    this.valueChange.emit(event.target.value);
  }
}
```

---

## Parent Component

```html
<app-child [(value)]="parentValue"></app-child>

<p>{{ parentValue }}</p>
```

Component:

```ts
parentValue = "Hello";
```

This works like ngModel.

---

# 3. Two-Way Binding with Signals (Modern Angular)

Angular now supports model inputs.

Example:

Child:

```ts
import { model } from '@angular/core';

name = model('');
```

Parent:

```html
<app-child [(name)]="parentName"></app-child>
```

This is cleaner than @Input/@Output.

Useful in signal-based architecture.

---

# ngModel Under the Hood

This:

```html
<input [(ngModel)]="name">
```

Internally becomes:

```html
<input [ngModel]="name" (ngModelChange)="name = $event">
```

And internally:

```html
<input [value]="name" (input)="name = $event.target.value">
```

Flow:

```text
ngModel → Property Binding + Event Binding
```

# FormsModule Requirement

Important:

If using ngModel:

Must import:

```ts
FormsModule
```

```text

```

# Two-Way Binding vs One-Way Binding

| One-Way Binding                      | Two-Way Binding  |
| ------------------------------------ | ---------------- |
| Single direction                     | Both directions  |
| Simpler                              | More interactive |
| Component → View OR View → Component | Component ↔ View |

Examples:

One-way:

```html
[value]="name"
```

Two-way:

```html
[(ngModel)]="name"
```

# Common Mistakes

## Forgetting FormsModule

Wrong:

```ts
imports: []
```

Correct:

```ts
imports: [FormsModule]
```

---

## Wrong Syntax

Wrong:

```html
(ngModel)="name"
```

Correct:

```html
[(ngModel)]="name"
```

---

## Using ngModel without property

Wrong:

```html
<input [(ngModel)]>
```

Correct:

```html
<input [(ngModel)]="name">
```

---

## Mixing reactive forms incorrectly

Avoid:

```html
<input [(ngModel)]="name" formControlName="name">
```

This can create conflicts.

---

# Best Practices

* Use ngModel for simple forms
* Use Reactive Forms for complex forms
* Use custom binding for reusable components
* Use signals for modern state-based components
* Always keep state predictable
